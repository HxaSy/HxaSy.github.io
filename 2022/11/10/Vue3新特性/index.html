一、创建一个 Vue 3 应用

Vue 2 中我们创建一个实例用到的是 Vue() 函数，而在 Vue 3 中则是通过 createApp()

<div id="app">
  
</div>
// Vue 2 
const app = new Vue({

})

app.mount('#app')
// Vue 3
import { createApp } from 'vue'

// 组件注册
const app = createApp({
  
})

// 组件挂载
app.mount('#app')

二、Vue 3 新特性

1. 组合式 API

1.1 概念

Vue 有多种 API 风格，全局 API，组合式 API 和选项式 API 等等，我们在 Vue 2 中使用的是选项式 API，有很多的逻辑点，比如 data()、watch()、computed()、method 等等。可当我们的逻辑点比较复杂时，想要修改部分代码就变得很麻烦，逻辑点过多就需要花更多的时间寻找，所以 Vue 3 中就新引入了组合式 API，把逻辑点组合在 setup() 中，让自己的项目有更好的逻辑复用，更灵活的代码组织。

组合式 API 是一系列 API 的集合，包含以下 API ：

  • 响应式 API : 响应式状态的 ref()、reactive()、监听器 watch()、watchEffect() 等等

  • 生命周期: 在 Vue 3 中，所有的声明周期都是以 on 开头，并且取消了 beforeCreate() 和 created() 这两个生命周期，组件卸载相关周期更名为 onBeforeUnmount() 与 onUnmounted()。

  • 依赖注入: 使我们可以在使用响应式 API 时，利用 Vue 的依赖注入系统。

1.2 基本使用

组合式 API 需要借助 setup() 钩子作为入口，使用响应式 API 来声明状态，以下的代码是在 SFC 下编写的，不需要 createApp()

<script>

// 响应式状态和生命周期都需要导入
import { ref , onMounted } from 'vue'

export default {

  // 逻辑编写
  setup() {
    const count = ref(0)

    function increment() {
      count.value++
    }

    // 变量或是函数都需要返回，供模板及其他选项使用
    return {
      count,
      increment
    }
  },

  // 通过组件实例访问
  onMounted() {
    console.log(this.count)
  }
}
</script>

<template>
  // 已经通过 ref 解析，无需添加 .value
  <button @click="increment">{{ count }}</button>
</template>

1.3 响应式 API

1.3.1 ref() 和 reactive()

使用 ref() 或者 reactive() 可以声明一个响应式的变量，具体使用方法如下：

import { ref, reactive, onMounted } from 'vue'

const message = ref('learn Vue 3')

const user = reactive({ name : 'wade' })

两个 API 的区别主要在于 ref() 能声明任何类型的数据，而 reactive() 只对 数组、Map、对象、Set 这类起作用。响应式 默认是深层式的，它会影响到所有嵌套的属性。一个响应式对象也将深层地解包任何 ref 属性，同时保持响应性。更多的细节请访问官方文档 https://cn.vuejs.org/api/reactivity-core.html

onMounted() {
  console.log(this.count)
  console.log(user.name)
}

<template>
  <div>{{ `${user.name} ${count}`}}</div>
</template>
1.3.2 计算属性 computed()

使用 computed() 来描述依赖响应式状态的复杂逻辑

import { reactive, computed } from 'vue'

setup() {
  const user1 = reactive({
    name : 'wade',
    jobs : [
      'teacher',
      'basketball player',
      'doctor'
    ]
  })
  
  // 一个计算属性 ref
  const result = computed( () => user.jobs.length > 0? 'yes' : 'no')

  return {
    user,
    result
  }
}

// 若直接访问响应式状态，在模板中多次引用的话，这样可看行就降低了
<p>wade have a job?</p>
<span>{{ user.jobs.length > 0 ? 'yes' : 'no' }}</span>

// 使用计算属性
<p>wade have a job?</p>
<span>{{ result > 0 ? 'yes' : 'no' }}</span>
1.3.3 监听器 watch() 与 watchEffect()

watch() 和 watchEffect() 的使用，其中 watch() 和在 Vue 2 中的用法是一样的

// watch() 可以监听一个或多个数据源的变化，并执行相关操作
// 数据源类型: ref 、响应式对象 、一个函数的返回值
/* 三个参数分别为: 数据源，回调函数(这个的参数为 数据变化时的新值、旧值以及清除副作用的函数)，
第三个参数为一个对象，可以设置深度监听等属性 */
const num = ref(0)

watch(
  num,
  (newValue , oldValue) => {

  },
  // 监听的数据源为一个对象，想要深度遍历时，设置 deep 为 true
  { deep : true }
)
// watchEffect
1.3.4 响应式 API 工具函数( isRef()、toRef()、toRefs())

isRef() 用于检测一个值是否为 ref ，可用作类型判断

const message = ref('hello')

const result = isRef(message) // true

toRef() 基于一个响应式的对象中的某个属性，创建一个 ref，与源对象保持同步，两者之一修改都会影响彼此

const player = reactive({
  name : ['wade', 'davis', 'paul'],
  num : 3
})

const numRef = toRef(player, 'num')

// 修改 numRef
numRef.value++
console.log(player.num) // 4

// 修改 num
player.num--
console.log(numRef.value) // 3

toRefs() 其实就是响应式对象里的所有属性调用了 toRef()

const music = reactive({
  name : '星晴',
  singer : 'jay',
  language : 'chinese'
})

// 将 music 转为普通对象，该对象属性指向源对象相应属性的 ref
const musicRef = toRefs(music)

music.name = '倒带'
// 需要通过 .value 访问
console.log(musicRef.name.value)
1.3.5 响应式 API 进阶函数( shallowRef()、toRow() )

shallowRef() 是 ref() 浅层式 ( 响应式默认是深层的 )

const obj = shallowRef({ 
  count : 1
})

obj.value.count = 2 // 不会响应

// 只有直接修改 .value 才会响应，这就是浅层的作用
obj.value = { count : 2 }

toRow() 返回一个代理的原始对象

const obj = {}

cosnt reactiveObj = reactive(obj)

console.log( toRow(reactiveObj) === obj ) // true

1.4. 生命周期

和 Vue 2 相比的话，Vue 3 就是将 Vue 2 中的 beforeDestroy 和 destroyed 名称分别变更成 beforeUnmount 和 unMounted， 然后用 setup 代替了两个钩子函数 beforeCreate 和 created，新增了 onRenderTracked 等用于调试的钩子。

1.5. 依赖注入( provide() 和 inject() )

父组件可以通过 provide() 向子组件传递值，子组件使用 inject() 来接收

// provide() 接受两个参数：
   // 1. 第一个参数是需要注入的 key，可以是一个字符串或者symbol 
   // 2. 第二个参数是需要传递的值，可以是一个静态的或者是响应式的
   
const num = ref(3)

provide('num' , num)
provide('count' , 3)
// 使用 inject() 接收父组件或者是整个应用(通过 app.provide() ) 传送的值

// 会遍历父组件链，寻找最近的一个
const num = inject('num')

// 响应式的
const count = inject('count')

// 若没有在父组件找到，可以设置默认值，不设置则返回 undefined
const message = inject('message', 'default msg')

2. <script setup> 语法糖

2.1 基本语法

<script setup> 只能在 SFC 中使用，是组合式 API 的语法糖

<script setup>
import { ref, onMounted } from 'vue'
import { increment } from './add.js'

const count = ref(0)

</script>

// 在语法糖中，import、变量、函数等可以直接在模板中使用，无需暴露
<template> 
  <div>{{ increment(count) }}</div>
</template>

2.2 引入组件

引入的组件也可以直接使用

<script setup>
import Child from './Child.vue'
</script>

<template>
  <Child />
</template>

2.3 自定义指令

在 <script setup> 中，自定义指令不需要显示注册，但是命名必须按照 vNameOfDirective 的格式

<script setup>

const vMyDirective = {
  beforeMounted(el) {
    // 需要对该元素执行的操作
  }
}

</script>

<template>
  <div v-my-directive>hello vue 3</div>
</template>

3. 组件通信

组合式 API 中 setup 函数一般接受两个参数，props 和 上下文，上下文中包含一些可能用到的值，比如 slots、emit、expose 等等，其组件的通信如下：

// 父组件

// 将该值传给子组件
const num = ref(13)

// :num num是子组件中的props
<Child :num="num"></Child>
// 子组件

// props 不能解构，不然会丢失响应性，上下文可以安全解构
export default {
  props : {
    num : Number
    default : 0,
    require : true
  }

  emits: ['add','average']

  setup(props, { emit }) {
    console.log(props.num)

    emit('average')
  }
}

在使用 <script setup> 的情况下，组件通信需要使用 defineEmits()、defineProps()、defineExpose() 等 API，具体使用如下：

// 父组件
// 这三个不需要导入，会与 setup 一起编译
import Child from './Child.vue'

const name = ref('davis')
const age = ref(29)

<template>
  // 与 Vue 2 中的用法一样
  <Child :name ="name" :age ="age" @increment="(val) => age += val"/>
</template>
// 子组件

// 接受从父组件的值，可以直接在模板中使用
const props = defineProps({
  name: {
    type: String,
    required: true
  },
  age: Number
})

// 设置 defineEmits() 传递值给父组件
const count = ref(3)

// 声明事件，返回一个处理函数(event, ...args) => instance.emit(event, ...args)
const emit = defineEmits('increment')

// 根据返回的函数，可以知道第一个参数是事件名称，第二个参数是传递给父组件的值
emit('increment',count)
*/

4. Teleport 传送组件

<Teleport> 是一个内置组件，把组件模板中的一部分 “传送” 到该组件 DOM 结构外层


// 在某个组件中
<div>Teleport 组件</div>

<!-- to 的值可以是一个 CSS 选择器字符串，也可以是一个 DOM 元素对象 -->
<Teleport to="body">
  <!-- 需要呈现的内容，可以在 body 中找到 a 标签 -->
  <a>click me</a> 
</Teleport>

5. 插槽

在 Vue 2 使用具名插槽时，会用到 slot，使用作用域插槽时，会用到 slot-scoped，但在 Vue 3 中进行了统一，只需要使用 v-slot:slotName，简写形式为 #slotName

<!-- 父组件 -->

<Child>
  <!-- 解构 message 内容 -->
  <template #slotA="{ message }">
    {{ message }}
    <!-- 此处是插槽的内容 -->
  </template>
</Child>

<!-- 子组件 -->
<div>
  <slot name="a" message"具名作用域插槽">
    <h1>插槽 A</h1>
  </slot>

  <slot name="b">
    <h1>插槽 B</h1>
  </slot>

  <slot name="c">
    <h1>插槽 C</h1>
  </slot>
</div>
